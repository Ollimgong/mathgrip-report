## 🗂️ 프로젝트 개요

Notion 월말평가 DB → 구글 시트 자동 동기화 → 브라우저에서 레포트 출력까지

- 서버 없음 (백엔드 없음)
- 설치 없음 (브라우저에서 바로 실행)
- **단일 HTML 파일** (CSS + JS 인라인 번들, ~127KB)
- **GitHub Pages** 무료 호스팅 → 링크 공유만 하면 끝
- Apps Script URL + 시트 ID 기본값 내장 → 첫 접속 시 설정 불필요

## ⚙️ Apps Script (노션 동기화 + 웹앱 통합)

구글 시트 → 확장 프로그램 → Apps Script → `apps_script_v2.js` 코드 붙여넣기

**시트 메뉴 기능 (`📊 매쓰그립`):**
- **📥 노션에서 가져오기** — "2026년 3월" 입력 → 해당 월만 동기화
- **🔄 현재 월 동기화** — 오늘 날짜 기준 현재 월 자동 동기화

**기술 상세:**
- 노션 API: 시행월(select) 필터 + page_size:10 페이징
- 504 타임아웃 시 2초 대기 후 자동 재시도
- 호출 간 350ms 간격 (API rate limit 방지)
- MASTER 수식 속성에서 직접 추출

**노션 Integration 주의:**
- 월말평가 DB + 학생 DB + 선생님 DB + 시험범위 DB 모두에 Integration 연결 필수
- relation으로 연결된 DB에 Integration이 없으면 해당 rollup/수식이 빈 값으로 나옴

**배포 (URL 유지):**
- 배포 → **배포 관리** → 기존 배포 연필(✏️) → 버전 "새 버전" → 배포
- ⚠️ "새 배포"를 하면 URL이 바뀜! 반드시 "배포 관리"에서 업데이트

## ⚠️ 알려진 제약사항

1. **노션 API 속도** — 수식/rollup이 많은 DB는 느림 (page_size:10으로 우회)
2. **노션 Integration** — relation 연결된 모든 DB에 Integration 필요
3. **Apps Script 재배포** — 코드 수정 시 "배포 관리"에서 새 버전으로 (URL 유지)
4. **로고** — 투명 배경 PNG 권장, localStorage 한도로 500KB 이하
5. **코멘트** — 400자 이내 권장, 초과 시 잘림
6. **설정(로고 등)** — 브라우저별 localStorage에 저장, 다른 브라우저에서는 재설정 필요
