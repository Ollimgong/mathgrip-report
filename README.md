## 🗂️ 프로젝트 개요

Notion에서 관리하는 월말평가 데이터를 **구글 시트**에 붙여넣기만 하면,  
버튼 하나로 레포트를 출력할 수 있는 **완전 정적 웹앱**.

- 서버 없음 (백엔드 없음)
- 설치 없음 (브라우저에서 바로 실행)

- 데이터 소스: 구글 시트 전용
- **단일 HTML 파일** (CSS + JS 전부 인라인 번들)

## ⚙️ Apps Script 코드

구글 시트 → 확장 프로그램 → Apps Script → 아래 코드 붙여넣기 → 배포

```javascript
function doGet(e) {
  const sheetId = e.parameter.sheet;
  const tabName = e.parameter.tab;
  const ss = SpreadsheetApp.openById(sheetId);
  if (!tabName || tabName === '__list__') {
    const names = ss.getSheets().map(s => s.getName());
    return ContentService
      .createTextOutput(JSON.stringify({ tabs: names }))
      .setMimeType(ContentService.MimeType.JSON);
  }
  const sheet = ss.getSheetByName(tabName);
  if (!sheet) {
    return ContentService
      .createTextOutput(JSON.stringify({ error: '탭을 찾을 수 없습니다: ' + tabName }))
      .setMimeType(ContentService.MimeType.JSON);
  }
  const data = sheet.getDataRange().getValues();
  const rows = data.map(row => row[0]).filter(v => v && String(v).trim());
  return ContentService
    .createTextOutput(rows.join('\n'))
    .setMimeType(ContentService.MimeType.TEXT);
}
```

**배포 설정:**
- 배포 → 새 배포 → 유형: **웹 앱**
- 다음 사용자로 실행: **나**
- 액세스 권한: **모든 사용자**

## 🔧 개발 가이드

### 번들링

```bash
python3 build.py
# → index_bundled.html 생성
# → 이것을 index.html로 교체하여 배포
```

`build.py`가 하는 일:
1. `css/style.css`를 `<style>` 태그로 인라인
2. `js/*.js` 5개 모듈의 `import/export` 제거 후 하나의 `<script>`로 합침
3. 모듈 간 중복 함수 자동 제거 (`parseMonthLabel`, `gradeOrder`, `getTeacherColor`)
4. `index.html`의 인라인 `<script type="module">`에서 다중 라인 import 제거

### 소스 수정 시 주의사항

- `js/` 파일 수정 후 반드시 `build.py` 재실행
- 함수 이름이 여러 파일에 중복되면 `build.py`의 `remove_duplicate_fn()` 목록에 추가 필요
- `type="module"` → 번들 시 일반 `<script>`로 변환되므로 전역 스코프 충돌 주의
